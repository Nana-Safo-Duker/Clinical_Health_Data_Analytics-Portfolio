# Contributing Guide

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature-name`
3. Make your changes
4. Commit your changes: `git commit -m 'Add some feature'`
5. Push to the branch: `git push origin feature-name`
6. Submit a pull request

## Code Style

### Python
- Follow PEP 8 style guide
- Use descriptive variable names
- Add comments for complex logic
- Include docstrings for functions

### R
- Follow the Tidyverse style guide
- Use descriptive variable names
- Add comments for complex logic
- Include documentation for functions

## Notebook Guidelines

- Clear markdown sections
- Explanatory text for each analysis step
- Code comments where needed
- Save outputs and visualizations

## Testing

- Test scripts with sample data before running on full dataset
- Verify that all visualizations are generated correctly
- Check that results are saved in the correct directories

## Documentation

- Update README.md if adding new features
- Document new functions and classes
- Update requirements.txt if adding new dependencies
